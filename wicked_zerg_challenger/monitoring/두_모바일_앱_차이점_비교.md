# 두 모바일 앱 디렉토리 차이점 비교

## 📋 개요

프로젝트에는 두 개의 모바일 앱 관련 디렉토리가 있습니다:
1. `mobile_app/` - 간단한 웹 기반 모니터링 페이지
2. `mobile_app_android/` - 완전한 Android 네이티브 앱

---

## 🔍 주요 차이점

### 1. **프로젝트 유형**

#### `mobile_app/`
- **유형**: 웹 기반 PWA (Progressive Web App)
- **기술 스택**: HTML, CSS, JavaScript
- **목적**: 간단한 웹 브라우저에서 접근 가능한 모니터링 대시보드
- **파일 수**: 1개 파일만 존재 (`public/index.html`)

#### `mobile_app_android/`
- **유형**: 네이티브 Android 앱
- **기술 스택**: Kotlin, Android SDK, Gradle
- **목적**: Android 기기에서 실행되는 완전한 모바일 앱
- **파일 수**: 392개 이상의 파일 (소스 코드, 리소스, 빌드 파일 등)

## 📱 사용 시나리오

### `mobile_app/` 사용 시나리오
- 빠른 웹 브라우저 접근이 필요한 경우
- 간단한 모니터링만 필요한 경우
- 별도 앱 설치 없이 접근하고 싶은 경우
- 크로스 플랫폼 (iOS, Android, Desktop) 지원이 필요한 경우

### `mobile_app_android/` 사용 시나리오
- 완전한 모바일 앱 경험이 필요한 경우
- Android 네이티브 기능을 활용하고 싶은 경우
- 오프라인 기능이 중요한 경우
- 푸시 알림 등 고급 기능이 필요한 경우
- Google Play Store에 배포하고 싶은 경우

## 🔧 개발 및 빌드

### `mobile_app/`
```bash
# 개발: 웹 서버에서 HTML 파일 제공
# 별도 빌드 과정 없음
# 브라우저에서 직접 열기 가능
```

### `mobile_app_android/`
```bash
# Android Studio에서 프로젝트 열기
# Gradle 빌드:
./gradlew assembleDebug  # 디버그 APK 생성
./gradlew assembleRelease # 릴리즈 APK 생성

# 또는 Android Studio에서:
# Build > Build Bundle(s) / APK(s) > Build APK(s)
```
